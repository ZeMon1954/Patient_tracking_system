# Skill: auto-document-bug

## Trigger
เรียกใช้ Skill นี้ทุกครั้งที่:
- แก้ไข bug หรือปัญหาความผิดพลาดในโค้ดสำเร็จ
- ผู้ใช้พูดว่า "จดจำบั๊กนี้ไว้", "บันทึกบั๊ก", "จำวิธีแก้ปัญหานี้ไว้หน่อย", "เรียนรู้บั๊กนี้ไว้"

## Instructions

> ⚡ **AUTO-EXECUTE**: เมื่อแก้บั๊กสำเร็จ หรือผู้ใช้สั่งให้จดจำ ให้ทำตามขั้นตอนต่อไปนี้ทันทีโดยไม่ต้องรอให้ผู้ใช้สั่งเพิ่ม

### Step 1 — วิเคราะห์และสรุปบั๊กที่เพิ่งแก้ไป
สรุปข้อมูลต่อไปนี้:
1. **Symptom (อาการ):** บั๊กแสดงอาการอย่างไร หรือเกิด Error message ว่าอะไร
2. **Root Cause (สาเหตุ):** ทำไมถึงเกิดปัญหานี้ขึ้น
3. **Solution (วิธีแก้ไข):** แก้ไขอย่างไร โค้ดที่ถูกต้องเป็นแบบไหน

### Step 2 — บันทึกลง Skill: known-bugs
ให้ทำการอัปเดตไฟล์ `c:\Users\mom20\OneDrive\Desktop\ฝึกงาน\Patient_tracking_system\.claude\skills\known-bugs\skill.md` (หากยังไม่มีโฟลเดอร์หรือไฟล์นี้ ให้สร้างขึ้นมาใหม่) โดยแทรกบั๊กใหม่ต่อท้ายรายการเดิม

**รูปแบบที่ต้องบันทึก:**
```markdown
### 🐛 [หัวข้อบั๊กสั้นๆ]
- **Symptom:** [อธิบายอาการ/Error]
- **Root Cause:** [อธิบายสาเหตุที่แท้จริง]
- **Solution:** [อธิบายวิธีแก้ หรือแนบ snippet โค้ดที่ถูกต้อง]
```

### Step 3 — แจ้งผู้ใช้
แจ้งให้ผู้ใช้ทราบสั้นๆ ว่า: "ผมได้เรียนรู้วิธีแก้บั๊กนี้และบันทึกลงในสกิล known-bugs เรียบร้อยแล้วครับ ครั้งหน้าถ้าเจอปัญหาคล้ายกัน ผมจะรู้วิธีแก้ทันที"
